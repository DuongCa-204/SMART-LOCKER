2. Tại sao phải dùng Thread

Ta muốn:

		UI vẫn chạy
		↓
		Gmail gửi ở nền

Giống như:

		Bạn ngồi quầy lễ tân

		Bạn = UI Thread

Nhân viên giao hàng = Worker Thread

Khách nói:

		Gửi OTP đi

Nếu bạn tự chạy đi gửi:

		Quầy trống
		Khách phải đợi

Nếu thuê nhân viên:

		Bạn vẫn ngồi quầy
		Nhân viên đi gửi

Đó chính là Thread.

4. Khi bấm SEND

Giả sử:

self.worker = SendOtpWorker(
    self.auth_service,
    email
)

Lúc này tạo ra object:

worker

chứa:

self.auth_service
self.email

Ví dụ:

email = "abc@gmail.com"

thì:

worker
 ├─ auth_service
 └─ abc@gmail.com
5. finished.connect()

Đoạn này:

self.worker.finished.connect(
    self.on_send_finished
)

nghĩa là:

Nếu worker làm xong
↓
Tự động gọi
on_send_finished()

Giống như:

Nhân viên giao hàng:

"Gửi xong rồi nhé!"
6. worker.start()

Đây là phần quan trọng nhất.

Khi:

self.worker.start()

PyQt tạo:

Main Thread
+
Worker Thread

Lúc này:

Main Thread

vẫn hoạt động:

Click
Touch
Animation
Video
Loading

Còn:

Worker Thread

chạy:

run()
7. Hàm run()

Trong file worker:

def run(self):

PyQt tự gọi hàm này.

Tương đương:

worker.start()

↓

worker.run()

Trong run:

success, message = (
    self.auth_service.send_otp_to_email(
        self.email
    )
)

Nó gửi Gmail.
